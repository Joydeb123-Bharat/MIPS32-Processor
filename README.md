# Custom 32-Bit Multi-Cycle MIPS Processor

## Overview
This project contains the complete RTL (Register-Transfer Level) design, simulation testbench, and software toolchain for a custom 32-bit Multi-Cycle MIPS processor. Designed from scratch in Verilog, the processor executes a customized instruction set architecture (ISA) using a 5-stage instruction cycle (Fetch, Decode, Execute, Memory, Write-Back) controlled by a Finite State Machine (FSM). 

The repository also includes a custom 2-pass Python Assembler to compile human-readable assembly mnemonics into binary machine code for the processor's instruction memory.

**Author:** Joydeb Sarkar | ECE, IIT Patna

---

## Architectural Features
* **Architecture:** 32-bit Data Bus, 32-bit Address Bus, Byte-Addressable Memory.
* **Control Unit:** Multi-Cycle FSM (Finite State Machine) minimizing hardware duplication.
* **Register File:** 32x32-bit multi-ported register bank (2 asynchronous reads, 1 synchronous write).
* **Memory:** Separate Instruction Memory (ROM) and Data Memory (RAM) blocks, each 4KB.
* **ALU:** Custom arithmetic logic unit with robust opcode decoding and branch target calculation.

## Supported Instruction Set Architecture (ISA)

### R-Type Instructions (Register-to-Register)
| Mnemonic | Opcode (6-bit) | Operation |
| :--- | :--- | :--- |
| `ADD` | `000000` | Addition ($R_d = R_s + R_t$) |
| `SUB` | `000001` | Subtraction ($R_d = R_s - R_t$) |
| `AND` | `000010` | Bitwise AND ($R_d = R_s \& R_t$) |
| `OR`  | `000011` | Bitwise OR ($R_d = R_s \| R_t$) |
| `SLT` | `000100` | Set on Less Than ($R_d = 1$ if $R_s < R_t$, else $0$) |
| `MUL` | `000101` | Multiplication ($R_d = R_s \times R_t$) |
| `HLT` | `111111` | Halt Processor Execution |

### I-Type Instructions (Immediate & Memory)
| Mnemonic | Opcode (6-bit) | Operation |
| :--- | :--- | :--- |
| `LW`  | `001000` | Load Word from Memory ($R_t = Mem[R_s + Imm]$) |
| `SW`  | `001001` | Store Word to Memory ($Mem[R_s + Imm] = R_t$) |
| `ADDI`| `001010` | Add Immediate ($R_t = R_s + Imm$) |
| `SUBI`| `001011` | Subtract Immediate ($R_t = R_s - Imm$) |
| `SLTI`| `001100` | Set Less Than Immediate ($R_t = 1$ if $R_s < Imm$) |
| `BNEQZ`| `001101`| Branch if Not Equal to Zero (If $R_s \neq 0$, $PC = NPC + Imm \times 4$) |
| `BEQZ`| `001110` | Branch if Equal to Zero (If $R_s = 0$, $PC = NPC + Imm \times 4$) |

## Repository Structure

### Hardware (Verilog)
* `MIPS32.v`: The top-level datapath and master FSM control unit.
* `ALU.v`: 32-bit Arithmetic Logic Unit.
* `RegisterBank.v`: 32x32-bit internal CPU storage.
* `InstructionMemory.v`: 4KB ROM for program instructions.
* `DataMemory.v`: 4KB RAM for variable storage.
* `tb_MIPS32.v`: The simulation testbench providing clock generation and execution monitoring.

### Software Toolchain (Python)
* `assembler.py`: A 2-pass Python compiler. Handles PC-relative branch offsets, label resolution, and binary generation.
* `program.txt`: (Example) Source assembly file.
* `imem_init.txt`: The compiled 32-bit machine code generated by the assembler (loaded by Vivado).
* `dmem_final_state.txt`: A memory dump generated by the testbench upon hitting the `HLT` instruction to verify program output.
